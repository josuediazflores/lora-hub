The idea

One base model. Many specialists. Swapped per turn.

Shipping a whole model for every task is wasteful. A LoRA adapter is only a few megabytes of difference. So LoRA Hub keeps one quantized base model loaded and slips tiny task-specific adapters in and out while you're still talking to it, in under a second.

Core mechanic

Hot-swap with per-turn provenance

The mlx-lm sidecar wraps the LoRA layers once and then just swaps the weights each turn. Nothing restarts.

A colored gutter down the side shows which adapter wrote each reply.
Swap markers appear between turns whenever the active adapter changes.
Drop in any HuggingFace adapter and it gets converted from PEFT format the first time you load it.

The storefront

An app store for adapters

Browse a catalog of community adapters, backed by Cloudflare. Hitting install streams the weights down to your machine, checks the SHA-256, and loads them straight into the running model. You never leave the app.

Tools & safety

Agentic, but on a leash

Every tool call is wrapped in an append-only audit log. The model can take action, but only inside bounds you can see and check.

The shell is allowlisted, with a read_only or standard preset on run_command.
Every URL the model fetches is run past an SSRF guard first.
Saving to memory pops an approval modal when you're on the 'ask' policy.
An audit viewer lets you replay exactly what the agent did.
